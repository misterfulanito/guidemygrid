<objective>
Retire the disproven Windows raw-copy installer and lock the retirement in place with an automated regression guard. This phase confirmed (per CONTEXT.md D-01) that Windows installs through the exact same `.ccx` + Creative Cloud Desktop mechanism as macOS — raw file-copy into `%APPDATA%\Adobe\UXP\PluginsStorage\...` never registers a plugin with Photoshop on any OS, an architectural property of Creative Cloud Desktop itself, not a macOS quirk. There is no custom Windows installer to build (WIN-01 is satisfied by the existing cross-platform `.ccx` builder), no install-time manifest to write (WIN-02 superseded, D-02), no "Photoshop is running" detection to add (WIN-03 superseded, D-02), and no custom uninstaller to register under HKEY_CURRENT_USER (WIN-04 superseded, D-05) — Creative Cloud Desktop owns the install and uninstall sequence end to end, leaving no hook for any of that custom behavior.

This plan does two things:
1. Deletes the five retired raw-copy scripts (D-03) and, in the same change, removes the `scripts/package.js` build step that copies them — without that fix, deleting the scripts makes `npm run package` crash with ENOENT on the next release (RESEARCH Pitfall 1).
2. Adds a cross-platform Jest regression test that fails if any retired script reappears, if `scripts/package.js` starts referencing one again, or if `manifest.json` regains a `requiredPermissions` block (the property that reintroduces the admin-password prompt WIN-01 exists to avoid).

No new install/uninstall/manifest logic is written (explicitly foreclosed by D-02/D-05), and no new npm packages are added (RESEARCH confirmed none are needed). No migration/cleanup of old raw-copy installs is required — D-04 confirmed via real download data (1 total download of the old `-installer.zip` across v1.6.0–v1.6.2) that the retired installer was effectively never used, and the retired scripts write no registry state to clean up (RESEARCH Runtime State Inventory, Assumption A3).

Purpose: Remove dead, broken installer code and its now-broken packaging dependency, and prevent both from silently returning.
Output: A `distribution/photoshop/windows/` directory with no installer scripts left, a `scripts/package.js` that only builds the `.ccx`, and a regression test that keeps it that way.
</objective>
