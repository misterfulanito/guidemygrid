<objective>
Give macOS — the project's primary platform — the same build-artifact regression guard Windows already has. Per D-09, since Creative Cloud Desktop cannot be driven headlessly in CI (confirmed Phase 2, D-06) and there is no custom install/uninstall code left to filesystem-diff (D-01/D-02), INTEG-03's "automated install→uninstall filesystem-diff check" is rescoped to a build-artifact regression check: build the `.ccx` the way a real release does, then assert its contents have no `requiredPermissions` (which would trigger CC Desktop's admin prompt — see 01-RESEARCH.md) and contain none of the retired installer/uninstaller scripts.

Per D-10, add a new `macos-latest` CI job mirroring the existing `.github/workflows/windows-ccx-verify.yml` almost exactly, using `unzip` (macOS built-in) + `node -e` for JSON parsing instead of PowerShell's native zip/JSON handling — avoiding a `jq` dependency the project doesn't otherwise use. Per D-11, extend the retired-script list to include the two macOS uninstaller scripts (`build-mac-uninstaller.js`, `uninstall-preinstall`) so the guard covers this phase's retirement too.

Real device-level uninstall-residue confirmation is deferred (D-12), mirroring Phase 2's Windows precedent — not blocking. This CI check is the automatable proxy for "nothing left behind," complementary to Plan 03-01's repo-level Jest guard (they catch different failure modes — Pitfall 3).

Purpose: Catch a `requiredPermissions` re-add or a retired-script leak in the built macOS artifact before release.
Output: `.github/workflows/macos-ccx-verify.yml`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → CI runner | `macos-latest` GitHub-hosted VM builds and inspects the `.ccx`; runs with GitHub-provisioned ephemeral credentials, no write scope needed |
