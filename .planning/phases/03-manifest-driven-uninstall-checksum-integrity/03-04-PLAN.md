<objective>
Complete the INTEG-04 security review of everything that still runs during `npm run package`/`publish:*` and in CI, then lock the findings in as both an automated regression test and a short written summary (D-15).

Per D-13, the review scope is the remaining release/build scripts: `distribution/photoshop/build-ccx.js`, `scripts/package.js`, `release/version.js`, `release/github-release.js`, and the new `release/checksums.js` — there is no custom installer/uninstaller script left on either platform (retired in Plan 03-01). The review must apply differentiated severity (Pitfall 4, A3): these scripts run only on the developer's own machine or an ephemeral GitHub CI runner — a fundamentally lower-risk context than the retired root-elevated, end-user-facing installer this milestone eliminated. Bare command names (`git`/`gh`/`npm`) are a real-but-low finding, not equivalent to the installer's remote PATH-hijack bug. The review applies exactly one concrete, cheap hardening — a `zip`-availability preflight in `build-ccx.js` (the highest-value target: it's already a known cross-platform pain point per the pending todo) — and cleans up the dead `installer.zip` reference in `release.yml`, documenting the rest as accepted low risk.

ASVS Level 1: no auth/session/access-control surface exists in this phase; the applicable controls are V6 Cryptography (Plan 03-02) and Tampering/EoP hygiene on build/CI tooling.

Purpose: Prove INTEG-04's env/PATH-trust review was done, with honest severity framing, and guard the one remediation.
Output: hardened `build-ccx.js`, cleaned `release.yml`, `release-script-safety.test.ts`, `03-SECURITY-REVIEW.md`.
</objective>
