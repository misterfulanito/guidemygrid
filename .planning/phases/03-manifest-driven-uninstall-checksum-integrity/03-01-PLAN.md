<objective>
Retire the last vestige of custom uninstall code on macOS and document the real uninstall mechanism. Per D-01/D-02, Creative Cloud Desktop already owns install AND uninstall end-to-end for the `.ccx`-distributed plugin (established Phase 1/2) — there is no manifest to attach a custom uninstaller to. INTEG-01 is therefore satisfied by deletion + documentation + a regression guard, not by new tracking code.

This plan deletes `scripts/build-mac-uninstaller.js` and `scripts/pkg-resources/uninstall-preinstall` (the root-requiring `pkgbuild --install-location /` uninstaller that directly contradicts this milestone's "no admin/root" goal), removes their invocation and staging from `scripts/package.js`, documents CC Desktop's "Manage Plugins" as the uninstall path (D-03), and adds a Jest regression test (D-11) mirroring the existing Phase 2 `installer-retirement.test.ts`.

No legacy-cleanup path is built (D-04): the user confirmed they are the only person who has ever installed GuideMyGrid, so no real end user carries orphaned pre-`.ccx` files. Real device-level uninstall-residue confirmation is deferred (D-12) — not blocking this phase.

Purpose: Eliminate the final root-requiring code path in the project and prevent its silent return.
Output: Two deleted scripts, a cleaned `package.js`, a README uninstall note, and a regression test.
</objective>
