<objective>
Build the non-interactive core of the macOS installer as a directly-testable shell routine, plus the Jest validation harness the phase's Nyquist gates require. `install-payload.sh` performs the file copy into `~/Library/Application Support/Adobe/UXP/PluginsStorage/PHSP/<ver>/Plugin/com.guidemygrid.plugin/` (MAC-01, user-level, no elevation) and writes an install-time manifest listing every path it created (MAC-02), using only absolute binary paths and an explicit clean PATH (MAC-04).

Why a separate shell routine instead of inline AppleScript: the full `.app` (Plan 04) shows blocking GUI dialogs, so it cannot run in a headless test. Extracting the copy+manifest logic into `install-payload.sh` makes MAC-01/MAC-02/MAC-04 automatically verifiable and gives Plan 04's AppleScript a clean interface to call.

Purpose: Deliver the verifiable, unprivileged copy+manifest core and its automated proof.
Output: `install-payload.sh`, the two Jest test files, and the `tsconfig.jest.json` that lets Jest compile tests living under `distribution/`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer process â†’ user filesystem (~/Library/.../PluginsStorage) | The routine writes plugin files and a manifest into the user's home tree; path resolution and command lookup are the sensitive surfaces |
| inherited shell environment ($PATH, rc files) â†’ install-payload.sh | An attacker-controlled PATH or rc file could hijack a bare command name |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-06 | Tampering | PATH hijacking via bare command names in install-payload.sh | high | mitigate | Explicit `PATH=/usr/bin:/bin` plus absolute-path binaries; no rc sourcing; enforced by installer-static.test.ts (MAC-04) |
| T-01-07 | Elevation of Privilege | Any accidental use of a package/system installer or `sudo` in the copy routine | high | mitigate | Pure user-level `/bin/cp`/`/bin/mkdir`; static scan asserts no `pkgbuild`/`productbuild`/`sudo` token present (MAC-01) |
| T-01-08 | Tampering | Path traversal / non-absolute destination writing outside the intended tree | medium | mitigate | Validate base is absolute and free of `..` before any write (Security V5) |
| T-01-09 | Information Disclosure | World-writable installed files or manifest readable/writable by other users | medium | mitigate | `chmod go-w` on created paths; manifest written to a GuideMyGrid-owned app-support dir the plugin runtime never reads (Security V12) |
| T-01-10 | Tampering (TOCTOU) | Race between staging and final placement | low | mitigate | Copy directly into place in the unprivileged process; if any temp is used, create it with absolute-path `/usr/bin/mktemp -d` |
</threat_model>
