<objective>
Wrap the Plan 03 copy+manifest core in a double-clickable, unprivileged macOS installer and its DMG, implementing the locked UX decisions and the Photoshop-running guard, then prove end-to-end that it never asks for an admin password.

- `installer.applescript`: confirmation dialog "Install GuideMyGrid?" (D-02) titled "Install GuideMyGrid" (D-03), a HARD block if Photoshop is running (D-04, MAC-03), a call into `install-payload.sh` (Plan 03) using an absolute-path shell invocation (MAC-04), and a success dialog "Installed! Open Photoshop" (D-05).
- `build-installer.js`: compiles the AppleScript to an unprivileged `.app` via `osacompile`, embeds the built `dist/` payload and `install-payload.sh` in the app bundle, sets the app's Finder icon from the existing plugin icon (D-06 — reuse, do not design), ad-hoc signs with `codesign --sign -`, and wraps it in a DMG with `create-dmg --no-code-sign` (MAC-01 packaging). It replaces the legacy root-requiring `.pkg` build.

SCOPE NOTE (surfaced, not silently decided): D-04 says the quit-Photoshop hard block applies to "both install and uninstall." This phase delivers the INSTALL side only. The uninstall side lands in Phase 3 with the manifest-driven uninstaller (INTEG-01), per the REQUIREMENTS.md traceability (Phase 1 owns MAC-01..04, none of which is the uninstaller). The merged-in legacy macOS uninstaller (scripts/build-mac-uninstaller.js, scripts/pkg-resources/uninstall-preinstall) remains root-requiring and manifest-blind until Phase 3 — this is expected, documented technical debt, not a Phase 1 regression.

Purpose: Ship the real, trustworthy, no-password macOS installer.
Output: installer.applescript, build-installer.js, a built .app + .dmg, and a verified no-elevation install flow.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user double-click (quarantined app) → installer execution | Gatekeeper evaluates the ad-hoc-signed app before it runs as the invoking user |
| installer .app → user filesystem | Unprivileged copy into ~/Library; no elevation |
| npm registry → build machine (create-dmg) | Supply-chain surface for the one new devDependency |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-11 | Elevation of Privilege | installer packaging path | high | mitigate | osacompile `.app` runs as the invoking user and structurally cannot elevate; legacy `.pkg` build retired; Task 3 confirms zero password prompt (MAC-01) |
| T-01-12 | Tampering | bare command names inside AppleScript `do shell script` calls | high | mitigate | Every `do shell script` begins with an absolute path; enforced by installer-static.test.ts (MAC-04) |
| T-01-13 | Spoofing | a renamed/fake process passing the "is Photoshop running" check | low | accept | This is a UX safety guard against accidental partial installs, not a security boundary; documented as such per RESEARCH.md Security Domain |
| T-01-SC | Tampering | supply chain of the `create-dmg` npm devDependency | high | mitigate | RESEARCH.md Package Legitimacy Audit cleared create-dmg 8.1.0 (verdict OK, sindresorhus, no postinstall script); pinned via package-lock; no [ASSUMED]/[SUS] package so no blocking-human legitimacy checkpoint required |
