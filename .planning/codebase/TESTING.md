## Test Structure

**Suite Organization:**
```typescript
describe('generateSideGuide — canvas (no selection)', () => {
  const params = { containerWidth: 1920, containerHeight: 1080, offsetX: 0, offsetY: 0 };

  test('left → x=0, vertical', () => {
    expect(generateSideGuide('left', params)).toEqual({ position: 0, orientation: 'vertical' });
  });
  
  test('center-v → x=960, vertical', () => {
    expect(generateSideGuide('center-v', params)).toEqual({ position: 960, orientation: 'vertical' });
  });
});
```

**Suite Characteristics:**
- Use `describe()` blocks for grouping related tests
- Group tests by scenario (e.g., "canvas" vs "selection with offset")
- Define shared test data at suite level (e.g., `const params = ...`)
- Use `test()` for individual test cases (not `it()`)
- Test names describe the input and expected output in arrow notation

**Patterns:**
- **Setup**: Test data defined in suite scope for reuse
- **No teardown**: Tests use pure functions, no cleanup needed
- **Assertions**: Simple `expect().toEqual()` for object equality

## Mocking

**What to Mock:**
- Currently: Nothing is mocked in existing tests
- No external dependencies in `gridGenerator.ts` (pure math functions)
- No Photoshop API calls in tested functions

**What NOT to Mock:**
- Pure utility functions tested directly without mocks
- Math operations tested with direct input/output
- Tests remain simple and focused on function behavior

**Future Mocking Guidance:**
If testing functions that call Photoshop API or external services:
- Could use Jest mocks for `photoshop` module (e.g., `jest.mock('photoshop')`)
- Could mock `photoshopBridge` service in component tests
- Currently not required due to thin test coverage

- Plugin cannot be tested in Node.js environment (requires Photoshop)
- `photoshopBridge.ts` depends on Photoshop API (cannot unit test in isolation)
- Currently externalized: `externals: { photoshop: 'commonjs2 photoshop' }`

**UI Component Testing:**
- No React Testing Library or similar framework installed
- Component tests would need ability to render UXP components
- Current coverage: 0 component tests

**Service Testing:**
- `updateChecker.ts` makes external HTTP requests (GitHub API)
- Requires mocking to test without network
- No mock setup observed in existing tests

**Integration Testing:**
- Would require Photoshop runtime with UXP enabled
- Not practical in CI/CD pipeline
- Manual testing is current approach
