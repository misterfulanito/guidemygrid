# Issue 004: Button UI standardization — native Adobe system

## Context

GuideMyGrid has at least two button styles that evolved independently: the Side Guides bar buttons and the main action buttons (Apply Guides, Save Preset, etc.). States (hover, active, disabled, loading) and visual properties (border, radius, padding, color tokens) are defined per-component with no shared system.

## Goal

Define a unified button design system in `tokens.css` and apply it consistently across all interactive controls so the plugin looks native inside Adobe Photoshop.

## Scope

- Audit all button/interactive element styles across the plugin
- Define tokens in `tokens.css`: `--btn-radius`, `--btn-border`, `--btn-bg`, `--btn-bg-hover`, `--btn-bg-active`, `--btn-color`, `--btn-color-active`
- Apply tokens in `SideGuidesBar.module.css` and `GridPanel.module.css`
- Ensure states (hover, active, disabled, loading) are consistent everywhere

## Files

- `src/styles/tokens.css`
- `src/components/SideGuidesBar/SideGuidesBar.module.css`
- `src/components/ColumnGrid/GridPanel.module.css`

## Acceptance Criteria

- [ ] All button-like elements use shared CSS tokens for colors and radius
- [ ] Hover, active, disabled, and loading states are visually consistent
- [ ] No hardcoded color values in component CSS files (use tokens)
- [ ] Plugin looks cohesive and native to the Adobe panel aesthetic
- [ ] Build passes with no errors
