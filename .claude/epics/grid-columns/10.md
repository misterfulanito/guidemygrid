# Task 009: Release packaging — .ccx build + GitHub Release + version sync

## Descripción

Preparar y publicar la primera release del plugin. Al terminar, cualquier usuario puede descargar el `.ccx` desde GitHub e instalarlo con doble-clic.

## Checklist de Release (ejecutar en orden)

### 1. Sincronizar versiones

Verificar que los tres archivos tengan el mismo número de versión (`1.0.0`):

```bash
# manifest.json
grep '"version"' manifest.json
# package.json
grep '"version"' package.json
# updateChecker.ts
grep 'PLUGIN_VERSION' src/services/updateChecker.ts
```

Si no coinciden, actualizar los tres antes de continuar.

También verificar que `GITHUB_REPO` en `updateChecker.ts` tenga el usuario/org real (no `{owner}`).

### 2. Build de producción

```bash
npm run build
```

Verificar que `dist/` contiene:
- `dist/index.js`
- `dist/index.html`
- `dist/manifest.json`
- `dist/icons/` (si los iconos se copian via webpack)

### 3. Empaquetar .ccx

```bash
cd dist
zip -r ../guidemygrid-v1.0.0.ccx .
cd ..
```

Verificar el contenido del .ccx:
```bash
unzip -l guidemygrid-v1.0.0.ccx
# Debe mostrar: index.js, index.html, manifest.json, icons/
```

### 4. Test de instalación manual

Antes de publicar:
- [ ] Doble-clic en `guidemygrid-v1.0.0.ccx`
- [ ] CC Desktop muestra diálogo de confirmación de instalación
- [ ] Aceptar → plugin disponible en Photoshop → Plugins → GuideMyGrid
- [ ] Verificar que el plugin instalado funciona igual que en desarrollo

### 5. Preparar iconos finales (si son placeholders)

El plugin necesita iconos de buena calidad para la listing de GitHub:
- `icons/icon-23.png` — 23×23 px (logo simple)
- `icons/icon-96.png` — 96×96 px (logo para spotlight)

Si se usaron placeholders durante desarrollo, crear los iconos reales antes del release.

### 6. Crear GitHub Release

```bash
# Crear tag
git tag v1.0.0
git push origin v1.0.0

# Crear release (via GitHub UI o gh CLI)
gh release create v1.0.0 \
  guidemygrid-v1.0.0.ccx \
  --title "GuideMyGrid v1.0.0" \
  --notes "$(cat << 'EOF'
## Primer release — MVP

### Qué incluye
- Grid de columnas configurable (1-12 columnas, gutter, márgenes)
- Presets de columnas con persistencia (hasta 3 presets gratis)
- Modo Replace / Add para guías existentes
- Preset built-in: "12-col Web Standard" (Bootstrap/Material)
- Compatibilidad: PS 22.0+, macOS y Windows

### Instalación
1. Descarga `guidemygrid-v1.0.0.ccx`
2. Doble-clic en el archivo
3. Creative Cloud Desktop instalará el plugin automáticamente
4. Abre Photoshop → Plugins → GuideMyGrid
EOF
)"
```

### 7. Verificar update checker post-release

Después de crear el release en GitHub:
1. Bajar `PLUGIN_VERSION` a `0.9.9` en `updateChecker.ts` temporalmente
2. Compilar y cargar en PS
3. Verificar que aparece `UpdateBanner` con versión `1.0.0` y link de descarga correcto
4. Restaurar `PLUGIN_VERSION` a `1.0.0`
5. Recompilar (build final de producción)

### 8. Página de descarga (mínima)

Para el MVP: el `README.md` del repositorio GitHub sirve como página de descarga. Asegurarse de que incluya:
- Descripción del plugin
- Captura de pantalla del panel
- Link directo a la última release (o badge de "Latest Release")
- Instrucciones de instalación (3 pasos)
- Requisitos: PS 22.0+, Creative Cloud Desktop

## Acceptance Criteria

- [ ] `PLUGIN_VERSION` en `updateChecker.ts` = `version` en `manifest.json` = `version` en `package.json` = `1.0.0`

- [ ] `guidemygrid-v1.0.0.ccx` se instala con doble-clic sin errores
- [ ] GitHub Release `v1.0.0` existe con el `.ccx` como asset

- [ ] `README.md` tiene instrucciones de instalación claras
- [ ] El plugin instalado desde `.ccx` es funcionalmente idéntico al de desarrollo
