# Task 003: Persistence — presetStorage + Zustand stores

## Descripción

Implementar la capa de persistencia (`presetStorage.ts`) y los tres stores de Zustand (`gridStore`, `presetsStore`, `uiStore`). Al terminar, el estado del plugin puede guardarse y recuperarse entre sesiones de PS.

## Archivos a crear

```
src/
├── services/
│   └── presetStorage.ts
└── store/
    ├── index.ts
    ├── gridStore.ts
    ├── presetsStore.ts
    └── uiStore.ts
```

## Implementación

### src/services/presetStorage.ts — Copiar directo
Copiar de `docs/reference/technical-decisions.md` → Sección 10.2.

Incluye:
- `STORAGE_KEY = 'guidemygrid_presets'`
- `CURRENT_VERSION = 1`
- `presetStorage.load()` con `try/catch` y fallback a `builtInPresets`
- `presetStorage.save()` que filtra presets built-in antes de guardar
- `builtInPresets`: "12-col Web Standard" y "8pt Grid" (ver Sección 10.2 para config exacta)
- `migrate()`: actualmente solo retorna `builtInPresets` para versiones desconocidas

**Test manual:** Guardar un preset, cerrar/reabrir PS, verificar que persiste.

## Reglas de los Stores (NO violar)

- Los stores NO llaman a `photoshopBridge` directamente
- Los stores NO llaman a `localStorage` directamente (solo `presetStorage` lo hace)
- Los stores NO tienen lógica de cálculo (eso es `gridGenerator`)
- `presetsStore.applyPreset()` solo delega a `gridStore.loadFromPreset()` — no aplica a PS

## Acceptance Criteria

- [ ] `presetStorage.load()` devuelve los 2 built-in presets si localStorage está vacío
- [ ] `presetStorage.save()` persiste presets de usuario (no built-in) en localStorage
- [ ] Los built-in presets no cuentan en el array guardado (solo `isBuiltIn: false`)
- [ ] Si localStorage está corrupto, `presetStorage.load()` devuelve built-ins sin lanzar error
- [ ] `gridStore.setColumnConfig({ columns: 6 })` actualiza solo el campo `columns` (immutable)
- [ ] `presetsStore.addPreset('Mi preset', config)` agrega y persiste inmediatamente
- [ ] `presetsStore.deletePreset(id)` no puede eliminar presets `isBuiltIn: true`
- [ ] `npm run type-check` sin errores
