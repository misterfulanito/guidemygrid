# Task 007: Presets panel + UpdateBanner — gestión de presets y notificación de updates

## Implementación

### Presets.tsx

Wireframe de referencia en `docs/reference/technical-decisions.md` → Sección 9.4.

```
┌─────────────────────────────────────────┐
│  Mis Presets              [+ Guardar]   │
├─────────────────────────────────────────┤
│  🏷 12-col Web Standard       [▶] [✕]   │ ← built-in (sin ✕)
│  🏷 Mi Preset                 [▶] [✕]   │ ← usuario
│  🏷 Grid Marketing            [▶] [✕]   │ ← usuario
│                                         │
│  [2/3 presets usados] Free              │ ← si es Free
│  Para más presets, actualiza a Pro.     │
└─────────────────────────────────────────┘
```

**Flujo "Guardar preset":**
- Botón "+ Guardar" abre un input inline (no modal) con el nombre
- Al confirmar: `presetsStore.addPreset(name, gridStore.getState().config)`
- Si usuario Free ya tiene 3 presets: deshabilitar botón + mostrar `ProBadge`
- No permitir nombres duplicados

```tsx
export function Presets() {
  const { presets, addPreset, deletePreset, applyPreset } = usePresetsStore();
  const config = useGridStore((s) => s.config);
  const license = useLicense();
  const [savingName, setSavingName] = useState('');
  const [showSaveInput, setShowSaveInput] = useState(false);

  const userPresets = presets.filter((p) => !p.isBuiltIn);
  const canSave = gates.canSaveMoreThan3Presets(license, userPresets.length);

  const handleSave = () => {
    if (!savingName.trim()) return;
    addPreset(savingName.trim(), config);
    setSavingName('');
    setShowSaveInput(false);
  };

  return (
    <section>
      <header>
        <h3>Mis Presets</h3>
        {canSave ? (
          <sp-button size="s" onClick={() => setShowSaveInput(true)}>+ Guardar</sp-button>
        ) : (
          <ProBadge message="Límite de 3 presets Free alcanzado" />
        )}
      </header>

      {showSaveInput && (
        <div>
          <sp-textfield
            value={savingName}
            onInput={(e) => setSavingName(e.target.value)}
            placeholder="Nombre del preset"
          />
          <sp-button size="xs" onClick={handleSave}>Guardar</sp-button>
          <sp-button size="xs" variant="secondary" onClick={() => setShowSaveInput(false)}>Cancelar</sp-button>
        </div>
      )}

      <ul>
        {presets.map((preset) => (
          <PresetItem
            key={preset.id}
            preset={preset}
            onApply={() => applyPreset(preset.id)}
            onDelete={preset.isBuiltIn ? undefined : () => deletePreset(preset.id)}
          />
        ))}
      </ul>

      {license.tier === 'free' && (
        <p>{userPresets.length}/3 presets de usuario</p>
      )}
    </section>
  );
}
```

### PresetItem.tsx

```tsx
interface PresetItemProps {
  preset: Preset;
  onApply: () => void;
  onDelete?: () => void; // undefined = built-in, no mostrar botón
}

export function PresetItem({ preset, onApply, onDelete }: PresetItemProps) {
  return (
    <li>
      <span>{preset.name}</span>
      {preset.isBuiltIn && <span title="Preset incluido">⭐</span>}
      <sp-action-button onClick={onApply} title="Aplicar preset">▶</sp-action-button>
      {onDelete && (
        <sp-action-button onClick={onDelete} title="Eliminar preset">✕</sp-action-button>
      )}
    </li>
  );
}
```

**Nota:** Al aplicar un preset, este carga la config en `gridStore` automáticamente y el usuario debe ir al tab "Grid" y hacer clic en Apply para aplicar a PS. El preset en sí no aplica a PS directamente.

### UpdateBanner.tsx

Spec en `docs/reference/technical-decisions.md` → Sección 2.5.

```tsx
interface UpdateBannerProps {
  info: UpdateInfo;
  onDismiss: () => void;
}

export function UpdateBanner({ info, onDismiss }: UpdateBannerProps) {
  const openDownload = () => {
    // UXP soporta apertura de URLs externas en el browser del sistema
    require('uxp').shell.openExternal(info.downloadUrl);
  };

  return (
    <div role="alert" className={styles.banner}>
      <span>Nueva versión {info.latestVersion} disponible</span>
      <sp-button size="xs" onClick={openDownload}>Descargar</sp-button>
      <sp-action-button size="xs" onClick={onDismiss} aria-label="Cerrar">✕</sp-action-button>
    </div>
  );
}
```

El banner se muestra en la parte superior del panel, antes de los tabs. Se cierra al hacer clic en ✕ y no vuelve a aparecer hasta la próxima sesión de PS (state local en `App.tsx`, no persiste).
